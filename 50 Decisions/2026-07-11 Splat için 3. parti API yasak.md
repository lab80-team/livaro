# Splat için 3. parti API yasak (KIRI / Luma / Polycam)

## Bağlam
Gaussian Splat üretimi için seçenekler araştırılırken 3. parti API'ler (KIRI Engine, Luma, Polycam) masadaydı.

## Karar
**Self-hosted gsplat** (nerfstudio backend) kullanılır; KIRI Engine API başta olmak üzere 3. parti splat API'lerine kod veya referans **eklenmez**. Kurucu (11.07): "KIRI Engine API kullanmayacağız — KIRI ile ilgili hiçbir kod veya referans olmasın."

## Gerekçe
Livaro zaten ARKit/RoomPlan kamera pozları kaydediyor; kendi pipeline'ı ~$0.5/sahne maliyetle eğitebilir ve veri şirkette kalır. 3. parti API'ler pozları atıp kendi SfM'ini koşuyor.

## Sonuçlar (Consequences)
- KIRI **uygulaması** yalnızca kalite kıyas ölçütü olarak kullanıldı (13 Tem testinde KIRI düzgün sonuç verdi, kendi pipeline vermedi — bkz. [[Gaussian Splatting self-hosted gsplat]]).
- Splat pipeline şu an rafta ([[2026-07-14 Gaussian Splatting rafa kaldırıldı]]) ama bu kısıt geri dönülürse de geçerli.
